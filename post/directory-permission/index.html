Directory Permission in Linux

In Linux, every directory and file has permission for admin, group, and other.

To see that permission, write in terminal ls -l :

(base) yusuf@DESKTOP-R0DO23T:~$ ls -l
total 8
drwxr-xr-x 26 yusuf yusuf 4096 Jan 10 22:27 anaconda3
drwxr-xr-x  2 yusuf yusuf 4096 Jan 24 16:00 belajar_shell
-rw-r--r--  1 yusuf yusuf    0 Feb  7 16:08 coba.txt

In the left side of each directory or file you can see the permission. Here is the explanation of that :

Part 1 : d or - ⇒ d shows that it is directory and - shows that it is a file

Part 2 until Part 4 consist of 3 words : r,w,and x. r ⇒ read, w ⇒ write, and x ⇒ executable. Read and write is obvious. executable means that you can execute the file if it is executable file.

So, what is the different between part 2, 3, and 4? Part 2 is permission for admin, part 3 is permission for group, and part 4 is permission for other.

Here is example how to change permission for admin

chmod +rwx [nama file] //add permission rwx 
chmod -rwx [nama file] //delete permission rwx 
chmod +r [nama file] //add permission r 
dst...

if you want to change permission for group or other, just add g for group and o for other :

chmod g+rwx [nama file]
chmod o+rwx [nama file]
